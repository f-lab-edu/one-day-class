# **프로젝트 개요**
본 프로젝트에서는 원데이 클래스 서비스를 제공하는 시스템을 개발합니다. 원데이 클래스 서비스는 기본적으로 HOST가 클래스를 개설하면 GUEST가 가능한 일자에 예약(신청)하는 서비스입니다. 원데이 클래스란 하루 몇 시간 동안 일회성으로 이뤄지는 수업을 말하며 자신이 관심 있는 분야를 직접 선택해서 체험해볼 수 있다는 장점이 있습니다.

원데이 클래스 서비스를 이용하는 사용자는 GUEST, HOST, ADMIN으로 나뉩니다. GUEST와 HOST는 일반 회원으로 서비스를 이용하기 위해 회원가입이 필요하며, ADMIN은 관리자로 관리자 계정을 통해서 서비스를 관리하는 역할을 수행합니다. GUEST는 원데이 클래스 수강자로서 클래스 신청, 후기 작성, 피드 조회 등의 활동을 수행하며, HOST는 원데이 클래스 개설, 수강생 관리 등의 활동을 수행합니다.

# **USE CASE / 시나리오**

## **1. ACTOR : GUEST**
### **<USE CASE 1> GUEST 회원가입**
1. HOST로 가입할지 GUEST로 가입할지 선택하는 화면이 보여지고 GUEST로 가입을 선택한다.
2. 아이디를 입력 후 중복확인 버튼을 클릭한다.
   * 존재하는 아이디가 있으면 중복 아이디 존재 메세지를 응답한다.
   * 존재하는 아이디가 없으면 사용 가능 메세지를 응답한다.
3. 비밀번호와 검증 비밀번호를 입력한다.
4. 전화번호를 입력 후 인증번호 발송 버튼을 클릭한다.
5. 회원가입 시스템은 유저의 전화번호로 인증번호를 전송한다.
6. 유저는 3분 내에 수신한 인증번호를 입력하고 인증하기 버튼을 클릭한다.
7. 회원가입 버튼을 클릭하면 회원가입 시스템에서 다음 사항들을 체크한다.
    * 아이디 중복확인 여부를 확인한다.
    * 비밀번호와 검증 비밀번호가 일치하는지 확인한다.
    * 비밀번호 입력값을 검증한다.(영문+숫자+특수문자 조합 8자리 이상)
    * 전화번호 인증 여부를 확인한다.
8. 전부 만족 시에 GUEST 회원가입을 완료한다.

### **<USE CASE 2> 로그인**
1. 유저가 아이디와 비밀번호를 입력 후 로그인 버튼을 클릭한다.
2. 로그인 시스템에서는 입력한 아이디와 비밀번호로 회원여부를 검증한다.
3. 일치하는 아이디와 비밀번호가 있으면 로그인 성공으로, 없으면 실패로 응답한다.

### **<USE CASE 3> 비밀번호 수정**
1. 비밀번호를 입력 후 비밀번호 수정 버튼을 클릭한다.
2. 비밀번호가 일치하면 비밀번호를 수정할 수 있는 폼을 보여준다.
3. 바꿀 비밀번호와 검증 비밀번호를 입력하여 비밀번호 수정을 완료한다.

### **<USE CASE 4> 회원탈퇴**
1. 회원 탈퇴 버튼을 클릭한다.
2. 비밀번호를 입력한다.
3. 회원 상태를 `탈퇴중`으로 변경한다.
4. 주 1회 정해진 일자에 탈퇴 신청한 유저의 정보를 일괄 삭제한다. 
> 관련 정책 : 회원 탈퇴 신청 후 일주일이 지나지 않은 유저는 복구가 가능하다. 신청 후 일주일이 지나면 유저 데이터를 모두 삭제한다.

### **<USE CASE 5> 클래스 목록 조회**
1. 최상위 카테고리 목록(카테고리명)과 카테고리 대표 사진이 유저에게 보여진다.
2. 조회하고자 하는 카테고리를 선택한다.
3. 선택된 카테고리의 하위 카테고리(카테고리명)들을 보여 준다.
4. 최하위 카테고리를 선택할 때까지 상기 2~3번의 흐름을 반복한다.
5. 선택된 최하위 카테고리에 포함된 클래스들의 목록을 보여준다.
6. 화면에 나타난 상품목록들 중에서 조회하고자 하는 클래스를 선택해서 클래스의 상세
정보 화면으로 이동할 수 있다.

### **<USE CASE 6> 클래스 검색**
1. 클래스가 개설된 지역을 선택한다.
2. 카테고리를 선택한다.
3. 검색어를 입력하고 검색 버튼을 클릭한다.
4. 해당 검색어를 포함한 클래스 목록을 보여준다.

### **<USE CASE 7> 클래스 상세 조회**
1. 클래스 목록 조회나 클래스 검색 후 상세한 정보를 알고 싶은 클래스 하나를 선택한다.
2. 선택한 클래스의 상세정보(클래스명, 강사명, 클래스 설명, 가격, 평점 등)를 보여준다.

### **<USE CASE 8> 클래스 신청**
1. 클래스 상세 정보를 조회하고 신청 버튼을 클릭한다.
2. 신청 가능한 일자를 조회한다.
3. 신청하고싶은 일자를 선택하고 신청 버튼을 클릭한다.
4. 시스템은 GUEST의 클래스 신청 상태를 `확정`으로 변경한다.

### **<USE CASE 9> : 클래스 신청 취소**
1. GUEST가 신청한 클래스 목록을 보여준다.
2. 취소하고 싶은 클래스를 선택 후 취소 버튼을 클릭한다.

### **<USE CASE 10> 클래스 후기 조회**
1. 클래스 상세 정보를 조회하고 해당 클래스의 후기 조회 버튼을 클릭한다.
2. 해당 클래스를 수강한 GUEST들의 수강 후기 정보(GUEST ID(일부 마스킹처리), 수강 일자, 후기, 평점)가 나열된다.

### **<USE CASE 11> 클래스 후기 등록**
1. GUEST가 수강했던 클래스 내역을 조회한다.
2. 일반후기와 사진후기 중 하나를 선택한다.
3. 클래스 하나를 선택해서 후기를 작성한다.
4. 클래스 평점을 등록한다.
5. 사진후기 선택 시 사진을 첨부한다.
6. 클래스 후기 등록을 완료한다.

### **<USE CASE 12> 피드 조회, 좋아요, 댓글 작성**
1. 다른 GUEST들이 등록한 후기가 시간순으로 보여진다.
2. 후기 사진, 작성자 이름, 작성 시간, 수강한 클래스명, 후기, 좋아요 수, 댓글이 보여진다.
3. 좋아요 버튼을 눌러서 해당 후기를 추천한다.
4. 댓글을 작성한다.

## **2. ACTOR : HOST**

### **<USE CASE 1> HOST 회원가입**
1. HOST로 가입할지 GUEST로 가입할지 선택하는 화면이 보여지고 HOST로 가입을 선택한다.
2. 아이디를 입력 후 중복확인 버튼을 클릭한다.
   * 존재하는 아이디가 있으면 중복 아이디 존재 메세지를 응답한다.
   * 존재하는 아이디가 없으면 사용 가능 메세지를 응답한다.
3. 비밀번호와 검증 비밀번호를 입력한다.
4. 전화번호를 입력 후 인증번호 발송 버튼을 클릭한다.
5. 회원가입 시스템은 유저의 전화번호로 인증번호를 전송한다.
6. 유저는 3분 내에 수신한 인증번호를 입력하고 인증하기 버튼을 클릭한다.
7. HOST로 회원가입 시에는 추가로 해당 분야의 전문가임을 입증할 수 있는 서류를 첨부한다.
8. 회원가입 버튼을 클릭하면 회원가입 시스템에서 다음 사항들을 체크한다.
    * 아이디 중복확인 여부를 확인한다.
    * 비밀번호와 검증 비밀번호가 일치하는지 확인한다.
    * 비밀번호 입력값을 검증한다.(영문+숫자+특수문자 조합 8자리 이상)
    * 전화번호 인증 여부를 확인한다.
9. 전부 만족 시에 HOST를 `가입 심사 상태`로 MEMBER 테이블에 등록한다.(관리자 승인 이후 회원가입이 완료된다.)

### **<USE CASE 2> 클래스 개설**
1. 클래스 카테고리를 선택한다.(HOST 분야에 따라 선택할 수 있는 클래스 카테고리가 달라진다.)
2. 클래스 정보(클래스명, 클래스 설명, 가격, 신청 가능 일자, 수강 정원, 장소, 시간)를 입력한다.
3. 클래스 개설 버튼을 클릭한다.

### **<USE CASE 3> 클래스 수정**
1. 개설중인 클래스를 보여준다.
2. 클래스 정보(클래스명, 클래스 설명, 가격, 신청 가능 일자, 수강 정원, 장소, 시간)를 수정한다.
3. 해당 클래스를 신청한 GUEST들에게 클래스 수정 알림을 보낸다.

### **<USE CASE 4> 클래스 폐지**
1. 개설중인 클래스를 보여준다.
2. 클래스를 선택하여 폐지한다.
3. 해당 클래스를 신청한 GUEST들에게 클래스 폐지 알림을 보낸다.

### **<USE CASE 5> 수강생 통계 조회**
1. 개설한 클래스 목록을 보여준다.
2. 클래스를 선택하면 해당 클래스를 수강한 수강생 통계를 보여준다.

## **3. ACTOR : ADMIN**
### **<USE CASE 1> HOST 가입 승인**
1. HOST로 회원가입한 유저가 첨부한 서류를 확인한다.
2. 서류를 검증하여 HOST 가입을 승인한다.
3. HOST에게 가입 승인 알림을 전송한다.

### **<USE CASE 2> HOST 가입 거절**
1. HOST로 회원가입한 유저가 첨부한 서류를 확인한다.
2. 서류를 검증하여 HOST 가입을 거절한다.
3. HOST에게 가입 거절 알림을 전송한다.

### **<USE CASE 3> 카테고리 등록**
1. 카테고리 목록이 관리자에게 보여진다.
2. 등록하고자 하는 카테고리명, 카테고리 설명, 카테고리 대표 사진을 입력하고 등록 버튼을 클릭한다.
3. 카테고리명 중복 여부를 체크한다.
4. 신규 카테고리가 등록되고 이를 포함한 카테고리 목록이 관리자에게 보여진다.
